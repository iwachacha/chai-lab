# Chai Lab UI Design Reference

このディレクトリの画像群は、Chai Lab のUI実装における正式なデザイン参照物である。
厳密な完成仕様書ではなく、世界観、余白、情報階層、カード設計、配色、モバイル画面構成をそろえるための視覚的アンカーとして扱う。

> この画像群は Chai Lab のUI実装における参考イメージであり、
> 世界観・余白・情報階層・カード設計・配色・モバイル画面構成の基準として扱う。
> ただし、文言・データ・機能配置・個別要素は最終仕様ではなく、
> 実際の要件・設計・MVP範囲に合わせて調整可能とする。

## 位置づけ

Chai Lab は、一般的なレシピ共有アプリやSNS風UIではなく、個人用のチャイ研究ノートである。
この画像群は、実装時に画面ごとの流派が分かれたり、無難な汎用レシピUIへ寄ったり、余白と情報階層が崩れたりすることを防ぐために参照する。

ただし、画像は機能仕様そのものではない。現行フェーズの実装可否は `docs/mvp-scope-contract.md`、画面仕様は `docs/app-design.md` と `docs/screen-acceptance-criteria.md` を優先する。
比較、系譜、定番化、インサイト、マイページのようにv1 MVP外の要素が画像に含まれていても、MVP範囲外の機能や導線を先回りして実装しない。

## 最重要アンカー

`01-home-anchor.png` を最重要の世界観アンカーとして扱う。
この画像から特に読み取るものは次である。

- 温かいが甘すぎない配色
- 研究ノートとしての静かな雰囲気
- ゆったりした余白
- モバイルで読みやすい情報密度
- 主行動が明確なレイアウト
- カードごとの情報整理
- 試作ログ、比較候補、定番レシピへ自然につながる構造感
- チャイ研究特化アプリとしての空気感

他の画像は、この世界観を各画面へ展開した参考例として扱う。

## 固定するもの

- 世界観
- 余白の考え方
- 情報階層
- カードベースの整理方法
- 色の方向性
- 1カラム中心のモバイルUI
- 研究ノートらしい落ち着き
- 試作ログ中心の見せ方
- 比較、派生、定番化へつながる構造感

## 固定しないもの

- 文言の完全一致
- 数値やダミーデータ
- 各機能の最終配置
- MVP段階で未採用の要素
- 画像や写真の必須化
- 細かなアイコンや装飾の一致

## 実装時の注意点

UI実装では、画像を表面的に再現するのではなく、既存の実装ルールへ翻訳する。
既存の共通レイアウト、コンポーネント思想、デザイントークン、余白、角丸、文字サイズを優先し、画像ごとに見た目専用コンポーネントを乱立させない。

研究記録の主役は写真ではなく記録内容である。参考画像に写真が含まれていても、実装は写真がなくても成立する構成にする。
色や装飾より、セクション分け、タイトルと補足の強弱、カードのまとまり、主ボタンと副ボタンの差、一覧から詳細への視線誘導を優先する。

Chai Lab のUIでは、試作ログ、比較、派生、定番化、振り返りにつながる構造を大切にする。
一方で、SNS風の賑やかさ、カフェ広告のような過剰装飾、レシピ投稿サービス風のUI、写真前提の設計、汎用ダッシュボード風の冷たいUIへ寄せない。

画像と仕様がずれる場合は、次の優先順位で判断する。

1. 安全性
2. データ保全性
3. 権限・公開範囲の正しさ
4. プロダクトの芯との整合
5. 実装可能性
6. 保守性
7. 画像との一致度
